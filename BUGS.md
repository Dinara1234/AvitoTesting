# Баг-репорты API Avito (тестовый стенд)

## Окружение общее
- **API базовый URL:** `https://qa-internship.avito.com`
- **Версии API:** v1 (POST, GET /api/1/...), v2 (DELETE, GET /api/2/...)
- **Инструменты тестирования:** Java 20, Rest Assured 5.4.0, JUnit 5
- **Дата тестирования:** 5 апреля 2026 г.

---

### Баг №1
**Описание:** POST /api/1/item при успешном создании возвращает недокументированный формат ответа (поле `status` вместо полей `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`).  
**Шаги воспроизведения:**  
1. Отправить POST-запрос на `/api/1/item` с валидным телом, например:
```json
{"sellerID":123456,"name":"ContractTest","price":999,"statistics":{"likes":5,"viewCount":10,"contacts":2}}
```
2. Проверить тело ответа.  
**Фактический результат:** Код 200, тело ответа: `{"status": "Сохранили объявление - <uuid>"}`.  
**Ожидаемый результат:** Согласно Postman-коллекции (документации) ответ должен содержать объект с полями `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`.  
**Серьёзность:** Critical  
**Окружение:** API v1, https://qa-internship.avito.com

### Баг №2
**Описание:** POST /api/1/item с нулевыми значениями статистики (`likes: 0`, `viewCount: 0`, `contacts: 0`) возвращает 400 с сообщением «поле likes обязательно», хотя поле передано.  
**Шаги воспроизведения:**  
1. Отправить POST-запрос с телом:
```json
{
  "sellerID": 123456,
  "name": "A",
  "price": 1,
  "statistics": {
    "likes": 0,
    "viewCount": 0,
    "contacts": 0
  }
}
```
**Фактический результат:** Код 400, тело:
```json
{"result":{"message":"поле likes обязательно","messages":{}},"status":"400"}
```
**Ожидаемый результат:** Код 200, объявление создаётся со статистикой (0,0,0).  
**Серьёзность:** Major  
**Окружение:** API v1, https://qa-internship.avito.com

### Баг №9
**Описание:** GET /api/1/item/{id} с id, содержащим спецсимволы (например, `test/1`), возвращает 404 с телом `{"message":"route ... not found","code":400}`, что не соответствует задокументированной структуре 404.  
**Шаги воспроизведения:**  
1. Выполнить GET /api/1/item/test/1.  
**Фактический результат:** Код 404, тело: `{"message": "route /api/1/item/test/1 not found", "code": 400}`. Отсутствуют поля `result` и `status`.  
**Ожидаемый результат:** Согласно документации, при 404 должен быть JSON `{"result": "laborum", "status": "cillum enim eiusmod"}`.  
**Серьёзность:** Minor  
**Окружение:** API v1, https://qa-internship.avito.com

### Баг №13
**Описание:** GET /api/1/statistic/{id} после удаления объявления возвращает 200 и статистику, хотя должна возвращать 404.

1. Создать объявление.  
2. Удалить его через DELETE /api/2/item/{id}.

**Ожидаемый результат:** Код 404 (статистика удаляется вместе с объявлением).
